# Schema & Database Type Audit

Last updated: 2026-06-02

## Summary

AuraPoS now uses **PostgreSQL native `uuid`** for application-owned UUID identifiers in `shared/schema.ts`, including `tenants.id` and tenant foreign-key columns. Auth-provider tables in `apps/api/src/lib/auth-schema.ts` intentionally continue to use text IDs because Better Auth/session provider IDs are external string identifiers rather than AuraPoS domain UUIDs.

## Actual Database Audit Query

Run this query against the target database before applying UUID migrations. It audits every table represented by `shared/schema.ts` plus the migration-managed `cfd_devices` table that stores tenant-owned device records.

```sql
WITH expected(table_name) AS (
  VALUES
    ('users'),
    ('business_types'),
    ('tenants'),
    ('outlets'),
    ('user_outlet_assignments'),
    ('tables'),
    ('tenant_module_configs'),
    ('product_categories'),
    ('products'),
    ('outlet_product_configs'),
    ('product_option_groups'),
    ('product_options'),
    ('order_types'),
    ('tenant_order_types'),
    ('order_number_sequences'),
    ('orders'),
    ('order_items'),
    ('order_item_modifiers'),
    ('order_payments'),
    ('kitchen_tickets'),
    ('tenant_features'),
    ('terminals'),
    ('sync_batches'),
    ('sync_events'),
    ('server_sync_conflicts'),
    ('inventory_movements'),
    ('inventory_sync_errors'),
    ('cfd_devices')
)
SELECT
  c.table_schema,
  c.table_name,
  c.column_name,
  c.data_type,
  c.udt_name,
  c.character_maximum_length,
  c.is_nullable,
  c.column_default
FROM information_schema.columns c
JOIN expected e ON e.table_name = c.table_name
WHERE c.table_schema = 'public'
ORDER BY c.table_name, c.ordinal_position;
```

### Castability Preflight

Migration `0015_native_uuid_alignment.sql` includes this preflight pattern before each type change: values must match a UUID string regex before `ALTER COLUMN ... TYPE uuid USING ...::uuid` runs. For manual audit, this helper query finds non-castable values in one table/column:

```sql
SELECT '<table>' AS table_name, '<column>' AS column_name, <column>::text AS invalid_value
FROM <table>
WHERE <column> IS NOT NULL
  AND <column>::text !~* '^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$'
LIMIT 20;
```

## Standard Chosen

Chosen standard: **native PostgreSQL `uuid`**.

Rationale:

- Most entity identifiers in current Drizzle code were already modeled as `uuid(...).defaultRandom()`.
- `tenants.id` was generated by `gen_random_uuid()` while still declared as `varchar`, creating drift between semantic UUID values and physical column type.
- Native UUID FKs improve referential integrity, operator/index consistency, and schema clarity for tenant isolation.
- Drizzle still exposes UUID values as TypeScript strings, so API payloads and frontend usage remain string-compatible.

## Columns Converted in `shared/schema.ts`

The migration and schema update convert:

- Primary UUID IDs that were historically `varchar`: `tenants.id` and legacy `users.id`.
- Tenant references: all `tenant_id` columns in shared-schema tenant-owned tables.
- UUID references previously represented as varchar: `tables.current_order_id`, `order_items.variant_id`, `order_item_modifiers.option_group_id`, `order_item_modifiers.option_id`, `sync_events.batch_id`, and `server_sync_conflicts.server_order_id`.
- Migration-managed `cfd_devices.id` and `cfd_devices.tenant_id` to keep tenant-owned device records aligned with the native UUID tenant key.

## Migration Notes

`migrations/0015_native_uuid_alignment.sql` is explicit and defensive:

1. Enables `pgcrypto` for `gen_random_uuid()`.
2. Creates temporary helper functions.
3. Drops foreign keys on affected tables before type conversion.
4. Validates all non-null legacy values are UUID-shaped.
5. Casts columns with `USING column::uuid` and restores random UUID defaults where needed.
6. Recreates FKs with the original `ON DELETE` behavior used by the Drizzle schema/migrations.
7. Drops temporary helper functions.

If the migration fails with `Cannot cast ... to uuid`, fix or remap the invalid legacy value before rerunning. Do not bypass the preflight check because invalid tenant/order/product IDs can break tenant isolation and order/payment integrity.

## Drift Check / CI

- Local command: `pnpm db:check`
- CI command: `pnpm ci:db:check`
- Workflow: `.github/workflows/db-migrations.yml`

The command runs `drizzle-kit check --config drizzle.config.ts` and uses a harmless placeholder `DATABASE_URL` when no real database URL is set because Drizzle's check command validates migration metadata without opening a database connection.

## Environment Limitation for This Audit Batch

This container did not have `DATABASE_URL` set, so live `information_schema.columns` results could not be collected here. The audit SQL above must be run against staging/production before applying the migration to verify actual column types and catch non-UUID legacy values.
